## 📖 О проекте
**KairoMaestro** — это современная веб-платформа, созданная специально для организаторов свадеб и мероприятий, а также для сотрудников, работающих на этих событиях. Сервис объединяет все необходимые инструменты в одном месте: от подбора персонала до управления расписанием, от коммуникации до финансового учета.

Платформа решает ключевые задачи организационного процесса, автоматизируя рутинные операции и обеспечивая прозрачность взаимодействия между организаторами и командой.

## 🗄 Структура базы данных

Проект использует нормализованную реляционную базу данных, состоящую из **14 таблиц**:

### Основные таблицы

- **`users`** — пользователи системы (работники и организаторы)
- **`worker_profiles`** — расширенные профили работников со специализацией
- **`events`** — информация о мероприятиях
- **`event_staff`** — связь сотрудников с мероприятиями и их статусы
- **`worker_availability`** — управление доступностью работников
- **`chats`** — чаты (групповые и личные)
- **`messages`** — сообщения в чатах с поддержкой файлов
- **`payments`** — платежи и компенсация расходов
- **`notifications`** — система уведомлений
- **`action_logs`** — журнал действий организаторов
- **`worker_statistics`** — статистика работы сотрудников
- **`event_templates`** — шаблоны мероприятий
- **`event_media`** — медиафайлы мероприятий
- **`chat_participants`** — участники чатов

## 🔒 Безопасность

Платформа реализует современные стандарты безопасности:

- 🔐 **Хеширование паролей** — используется `bcrypt` с cost factor 10
- 🛡️ **Защита от SQL-инъекций** — все запросы через PDO prepared statements
- 🚫 **Защита от XSS** — автоматическая экранирование через функцию `escape()`
- 🍪 **Безопасные сессии** — HTTP-only cookies, lifetime 30 дней
- 🔒 **HTTPS** — принудительное перенаправление на защищенное соединение
- 📁 **Защита файлов** — конфигурационные файлы вне веб-корня

## 🚀 Ключевые функции

### 💬 Система чатов

- **Групповые чаты** — для каждого мероприятия автоматически создается групповой чат команды
- **Личные чаты** — приватное общение между организатором и работником
- **Поддержка файлов** — загрузка изображений, видео, PDF документов
- **Уведомления** — моментальные оповещения о новых сообщениях

### 💰 Система платежей

- **Автоматический расчет** — оплата за часы работы рассчитывается автоматически
- **Компенсация проезда** — учет расходов на такси или пробега собственного автомобиля
- **Загрузка чеков** — прикрепление подтверждающих документов
- **Статусы оплаты** — прозрачный процесс: ожидает → подтверждено → оплачено

### 📅 Календарь и доступность

- **Личный календарь** — работники видят все свои мероприятия в удобном формате
- **Отметка недоступности** — возможность указать дни, когда не работаете
- **Автоматическая проверка** — организаторы видят доступность при назначении

### 📊 Аналитика

- **Статистика работника** — общее количество мероприятий, заработок, часы работы
- **Статистика организатора** — количество созданных мероприятий, активные/завершенные
- **История платежей** — полная история всех выплат с фильтрацией по статусу

## 🔄 Workflow

### Для работника:

1. **Регистрация** → Указывает специальность, ставку, наличие автомобиля
2. **Профиль** → Загружает фото, заполняет информацию
3. **Календарь** → Отмечает доступность, просматривает назначенные мероприятия
4. **Мероприятие** → Получает приглашение, подтверждает участие
5. **Работа** → Отмечает прибытие, общается с командой
6. **Завершение** → Указывает отработанные часы, расходы на проезд
7. **Оплата** → Получает подтверждение и выплату

### Для организатора:

1. **Регистрация** → Создает аккаунт организатора
2. **База работников** → Просматривает доступных сотрудников
3. **Создание мероприятия** → Указывает детали, выбирает локацию
4. **Подбор команды** → Назначает работников с проверкой доступности
5. **Управление** → Следит за статусами, общается с командой
6. **Оплата** → Одобряет выплаты, контролирует расходы

## 📝 Установка и настройка

1. **Загрузите файлы** на веб-сервер с поддержкой PHP 8+
2. **Создайте базу данных** MySQL 8 и импортируйте `database.sql`
3. **Настройте подключение** в `config/database.php`
4. **Создайте директории** для загрузок: `uploads/avatars`, `uploads/chat`, `uploads/receipts`, `uploads/media`
5. **Настройте права доступа** (755 для директорий, 644 для файлов)
6. **Настройте .htaccess** для вашего хостинга
